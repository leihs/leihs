explain_sql()

perform this in the runtime environment of a managed application, to explain the sql statement(s) executed within a segment of a transaction sample. returns an array of explanations (which is an array of array of string from the explain query) Note this happens only for statements whose execution time exceeds a threshold (e.g. 500ms) and only within the slowest transaction in a report period, selected for shipment to RPM

[Source]

     # File vendor/plugins/newrelic_rpm/lib/newrelic/transaction_sample.rb, line 138
138:       def explain_sql        
139:         sql = params[:sql]
140:         return nil if sql.nil?
141:         statements = sql.split(";\n")
142:         explanations = []
143:         statements.each do |statement|
144:           if statement.split($;, 2)[0].upcase == 'SELECT'
145:             explain_resultset = []
146:             begin
147:               connection = NewRelic::TransactionSample.get_connection(params[:connection_config])    
148:               if connection
149:                 # The resultset type varies for different drivers.  Only thing you can count on is
150:                 # that it implements each.  Also: can't use select_rows because the native postgres
151:                 # driver doesn't know that method.
152:                 explain_resultset = connection.execute("EXPLAIN #{statement}") if connection
153:                 rows = []
154:                 # Note: we can't use map.
155:                 # Note: have to convert from native column element types to string so we can
156:                 # serialize.  Esp. for postgresql.
157:                 explain_resultset.each { | row | rows << row.map(&:to_s) }  # Can't use map.  Suck it up.
158:                 explanations << rows
159:                 # sleep for a very short period of time in order to yield to the main thread
160:                 # this is because a remote database call will likely hang the VM
161:                 sleep 0.05
162:               end
163:             rescue => e
164:               x = 1 # this is here so that code coverage knows we've entered this block
165:               # swallow failed attempts to run an explain.  One example of a failure is the
166:               # connection for the sql statement is to a different db than the default connection
167:               # specified in AR::Base
168:             end
169:           end
170:         end
171: 
172:         explanations
173:       end
